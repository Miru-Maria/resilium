System Overview

Resilium is a full-stack, multi-artifact monorepo application delivering personalised resilience assessment on web and mobile. The system is built as a pnpm workspace containing three deployed artifacts and two shared library packages.

1.1 Repository Structure

resilium/
├── artifacts/
│   ├── api-server/          # Express 5 REST API (Node.js 24)
│   ├── resilium/            # React + Vite web application
│   └── resilium-mobile/     # Expo (React Native) mobile application
├── lib/
│   ├── db/                  # Drizzle ORM schema + migrations
│   └── replit-auth-web/     # Shared Replit Auth hook (useAuth)
└── pnpm-workspace.yaml

05

Scoring Methodology

Important
All scoring is deterministic. No AI is involved in score calculation. The algorithm runs server-side in scoring.ts and is auditable in full.

5.1 Dimension Weights

Financial
25%
Income stability, savings runway, dependents
Skills
20%
Breadth of transferable skills across 6 categories
Health
15%
Health status and medical skill possession
Mobility
15%
Mobility level, housing type, dependency load
Psychological
15%
Mental Resilience Composite from 10-question assessment
Resources
10%
Emergency supplies, survival and financial skills

5.2 Mental Resilience Composite (MRC)

Ten Likert questions (1–5 scale) map to six sub-dimensions. Each is normalised to 0–100:

subScore = round(((avg − 1) / 4) × 100) MRC = round(mean of all 6 sub-scores) // equal weight (1/6 each) Pathway: growth → MRC ≥ 60 (challenge-oriented framing) compensation → MRC < 60 (stabilisation-first framing)

5.3 Psychological Modifier

On the compensation pathway, a cross-area modifier dampens Financial and Skills scores to reflect the compounding effect of psychological pressure:

modifier = 0.85 + (MRC / 60) × 0.15 // MRC = 0 → modifier = 0.85 (15% dampening) // MRC = 59 → modifier ≈ 0.998 (negligible dampening) // growth pathway → modifier = 1.0 (no dampening)

5.4 Overall Score Formula

overall = (financial × modifier × 0.25) + (health × 0.15) + (skills × modifier × 0.20) + (mobility × 0.15) + (psychological × 0.15) + (resources × 0.10) All scores clamped to [0, 100].
06

AI Pipeline

6.1 Model & Infrastructure

AI calls use OpenAI's gpt-5.2 model via Replit's AI Integrations proxy. The API key is provisioned automatically by the platform — the server never stores or exposes it.

6.2 Report Generation Flow

1. Score calculation (deterministic, ~1ms)
2. Prompt construction from scores + input data + MRC pathway
3. OpenAI API call (full response awaited)
4. Response parsed into structured JSON schema:
   - riskProfileSummary
   - topVulnerabilities  [{ area, description, severity }]
   - actionPlan          [{ timeframe, action, priority, area }]
   - scenarioSimulations [{ scenario, impact, mitigation }]
   - dailyHabits         [{ habit, rationale, difficulty }]
   - checklistsByArea    { area: [{ itemId, text, priority }] }
   - recommendedResources [{ title, url, category }]
5. Full report written to database
6. Report ID returned to client
   (malformed responses trigger retry, max 2 attempts)

6.3 Pathway-Aware Framing

PathwayAI Framing Strategy
Growth (MRC ≥ 60)Challenge-oriented; ambitious long-horizon goals; opportunity-focused language
Compensation (MRC < 60)Emotionally scaffolded; stability-first; short milestones; reassurance-forward

6.4 Scenario Simulations (Pro)

Separate AI call for each scenario. Input: full report context + scenario type. Output: per-dimension impact deltas, recovery timeline estimate, tailored action steps. Scenarios: job_loss, natural_disaster, health_crisis, relocation.

07

Authentication & Security

7.1 User Auth (Replit Auth / OIDC)

Replit-managed OpenID Connect with PKCE. On callback, user record is upserted into users table. Session cookie: HttpOnly, Secure, SameSite: lax, 24-hour TTL.

7.2 Admin Auth

Cookie-based session using ADMIN_USERNAME / ADMIN_PASSWORD secrets. All /api/admin/* routes protected by middleware. Sessions expire after 24 hours.

7.3 Paddle Webhook Verification

const signed = `${timestamp}:${rawBody}`;
const expected = createHmac("sha256", PADDLE_WEBHOOK_SECRET)
  .update(signed).digest("hex");
assert(expected === h1); // from Paddle-Signature header

7.4 Rate Limiting

Anonymous assessment submissions: 6 per minute per IP via express-rate-limit. Authenticated users bypass this limit.

08

Freemium Access Control

Free limit
FREE_LIMIT = 2 — two complete assessments available at no charge for all users.

8.1 Anonymous Users

Count tracked in localStorage (resilium_free_count). At 2, the assessment page shows a paywall screen. A counter ("X of 2 free assessments used") is displayed in the assessment header throughout.

8.2 Authenticated Non-Subscribers

On page load, two parallel calls: GET /subscription/status and GET /users/me/report-count. If !isPro && count ≥ 2, paywall shown. Gate fails open on API error.

8.3 Pro Subscribers

isPro: true (status active or cancel_scheduled) bypasses all assessment gates and unlocks the Scenario Simulations runner at /scenarios/:reportId.

Testing & Quality

8 built-in personas each run the full assessment pipeline. A second AI evaluator scores each resulting report on quality, relevance, and empathy (0–10). A cross-persona synthesis report is generated at run end. Live progress via SSE. Exportable as Markdown/PDF.
